# NVIDIA 子站 IA 设计原则

> 从 NVIDIA 实践中抽象的可迁移原则 · 供对标 Ascend / 其他厂商站点架构

---

## 原则 1 · 受众优先于产品线

**观察：** 同一产品在不同受众场景下分轨交付，而非在主站混杂所有内容。

**NVIDIA 实例：**

| 产品 | 试用 / 部署入口 | 集成文档 | 大众传播 |
|------|-----------------|----------|----------|
| NIM | `build.nvidia.com` | `docs.nvidia.com/nim/` | `blogs.nvidia.com` |
| CUDA | `developer.nvidia.com`（下载） | `docs.nvidia.com/cuda/` | 主站产品页 |
| NGC 容器 | `catalog.ngc.nvidia.com` | `docs.nvidia.com/ngc/` | — |

**可迁移要点：** 先问「这条内容给谁看」，再决定放哪个域；避免在品牌主站堆砌 API 参数或安装步骤。

---

## 原则 2 · 子域命名即职能

**观察：** URL 域名段本身携带 IA 信号，爬虫与 LLM 可在 **站点级** 做过滤，无需解析页面内容。

**NVIDIA 命名模式：**

| 域名前缀 | 隐含职能 | LLM 路由含义 |
|----------|----------|--------------|
| `developer.*` | 开发者工具与社区 | 技术向，次选源 |
| `docs.*` | 官方文档 | 权威首选 |
| `catalog.*` | 软件/模型目录 | 部署行动点 |
| `build.*` | API 试用与部署 | 推理/部署行动点 |
| `blogs.*` | 内容传播 | 背景，非规范 |
| `investor.*` | 资本市场 | 工程技术问答排除 |

**可迁移要点：** 新站点规划时，让域名/子域名称可预测；避免 `company.com/random-path/docs-v2/` 式隐藏。

---

## 原则 3 · 内容双轨但入口分离

**观察：** 面向不同深度的内容各自独立入口，避免 SEO 与信源噪声互相污染。

**NVIDIA 实例：**

| 内容类型 | 入口 | 深度 | 受众 |
|----------|------|------|------|
| 大众博客 | `blogs.nvidia.com` | 概念 / 案例 / 趋势 | 媒体、泛用户 |
| 技术博客 | `developer.nvidia.com/blog` | 工程实践、性能优化 | 开发者 |
| 官方文档 | `docs.nvidia.com` | API、安装、Release Notes | 开发者 |
| 新闻稿 | `nvidianews.nvidia.com` | 产品发布、公告 | 媒体、投资者 |

**可迁移要点：**

- 「博客」不等于单一站点——大众叙事与技术深度应分轨
- 技术博客可链向 docs，但不应替代 docs 成为 API 参考
- 对 LLM：路由规则应区分 `blogs.*`（低优先级）与 `docs.*`（高优先级）

---

## 原则 4 · 交付入口独立

**观察：** 「获取软件 / 部署服务」与「阅读说明」是两类用户意图，入口应分离。

**NVIDIA 实例：**

| 用户意图 | 入口 | 内容形态 |
|----------|------|----------|
| 拉取 NGC 容器 | `catalog.ngc.nvidia.com` | 镜像 tag、Helm Chart |
| 试用 NIM API | `build.nvidia.com` | 在线 Playground、部署向导 |
| 购买硬件/软件 | `marketplace.nvidia.com` | 商品页、报价 |
| 学习如何集成 | `docs.nvidia.com` | 分步指南、API 参考 |

**可迁移要点：**

- 目录/商城是 **行动点**（CTA），文档是 **说明层**；混在同一页面会增加抓取噪声
- LLM 回答「如何部署 X」时应优先路由到 docs + catalog/build，而非主站产品页

---

## 应用检查清单

在规划或评审站点 IA 时，可用以下问题自检：

1. **受众**：这个页面的 primary 读者是谁？是否放在了对应域？
2. **命名**：从 URL  alone 能否判断内容类型与可信度？
3. **双轨**：大众内容与技术规范是否入口分离？
4. **行动点**：「获取/部署」与「阅读/学习」是否指向不同域？
5. **LLM 路由**：技术问答是否有明确的优先域 / 次选域 / 排除域？
