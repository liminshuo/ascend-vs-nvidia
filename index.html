三条核心洞察

01

单域连贯 vs 多域专名——两种 IA 哲学

证据：昇腾文档 / 下载 / 论坛共用 hiascend.com 顶栏；NVIDIA 拆为 docs.* / developer.* / catalog.ngc.* 等专名子域。

结论：昇腾站内开发者回访更顺（少 ↗）；NVIDIA站外机器可读性更强（URL 即职能标签）。

02

受众覆盖决定 IA 宽度，非细节打磨

证据：消费者 / 投资者场景昇腾为「—」缺位；NVIDIA 有 GeForce / investor 独立链路。

结论：昇腾 IA 是聚焦开发者 + 企业的单域社区；NVIDIA 是全受众分流——差距在战略边界，非导航细节。

03

行业 / 决策类入口：标签与用户心智错位

证据：医疗 BD 找方案——昇腾须识别「解决方案→行业应用」；NVIDIA 顶栏一级 Industries→Healthcare。

结论：非开发者角色的最大摩擦来自 L0 标签不直接对应任务意图；开发者侧摩擦在深链 ID / 缺闭环向导。

IA 设计模式识别

模式 A · 单域社区型（昇腾）

  • 表现：文档 / 下载 / 论坛 / 市场共用 hiascend.com 顶栏
  • 优势：开发者回访零 ↗，上下文不丢
  • 代价：URL 语义弱，站外 / AI 难归类

模式 B · 子域专名型（NVIDIA）

  • 表现：docs.* / developer.* / catalog.ngc.* 按职能分域
  • 优势：外部引用与 SEO 信号清晰
  • 代价：跨站 ↗ 打断，需强互链图谱

模式 C · 三维 Mega Menu（NVIDIA 主站）

  • 表现：Products × Solutions × Industries 多路径触达
  • 优势：适配不同心智（产品 / 场景 / 行业）
  • 代价：信息重复，新手不知走哪条

模式 D · 开发者优先 L0（昇腾）

  • 表现：CANN 居 L0 首位，企业 / 行业次之
  • 优势：核心受众（开发者）路径最短
  • 代价：决策者 / 采购者 L0 线索弱

可执行建议 · So What

昇腾 · 优先改进
  1. 补站外信号：docs 子域或语义化 URL，不牺牲同域体验
  2. 环境向导：配套表 → 下载 → 镜像 串成一条路径（旅程阶段 1）
  3. 行业 L0 曝光：「解决方案 / 行业」提升可感知，对齐 BD mental model
  4. 迁移 / 调试闭环：错误码↔论坛互链；映射表 + 工具一站式
NVIDIA · 可借鉴处
  1. 子域专名体系：外部可发现性与权威归类
  2. Industries 一级入口：非开发者角色行业方案直达
  3. On-Demand / Research：内容子体系品牌化
  4. 代价意识：跨域 ↗ 已在旅程中量化，互链需维护

一句话总结：昇腾的 IA 是「聚焦开发者与企业的单域社区」——同域同顶栏让 B 端用户全程顺滑，但消费级、投资者等角色不在其服务范围内； NVIDIA 的 IA 是「全受众覆盖」——从消费者、玩家到投资者各有专属入口，以独立子域按角色分流。

可发现性对比分析

「可发现性」分两层：站内（用户已在官网，能否从导航找到目标）已在上方旅程与用户画像中逐项量化； 本节聚焦站外 / 机器可读——搜索引擎、AI 助手、第三方引用时，能否仅凭 URL / 域名判断「这是不是权威文档 / 工具 / 社区」。

判定逻辑：子域专名 > 路径语义 > 数字 ID 深链；同品牌内链图谱越完整，外部系统越容易把「权威中心」归到正确子站。

小结 · 对 IA 设计的启示

昇腾短板集中在站外可发现性——不是内容缺失，而是信号载体：同域路径 + 数字 ID + 外跳生态，让搜索引擎与 AI 难以像 NVIDIA 子域体系那样「一眼归类」。
